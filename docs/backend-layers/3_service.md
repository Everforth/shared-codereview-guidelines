## Service層（ビジネスロジック、データ処理 等）

### 必須チェック項目：

- 責務分離: Controller/Serviceの役割が適切に分離されているか
- DB操作: TypeORMクエリが最適化され、N+1問題を避けているか
- エラー処理: 適切なHTTPステータスコードを返し、意味のあるエラーメッセージを提供しているか
- セキュリティ: 認証・認可ガードが正しく設定されているか

### パフォーマンス考慮点：

- バッチ処理での一括操作
- 適切なキャッシュ戦略（TTL設定）
- 不要なDB呼び出しの削減

### 設計・命名規則：

- 特殊な実装には必ずコメントで理由を明記
- モジュール間で名前の衝突を避ける
