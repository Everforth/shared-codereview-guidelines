## Service層（ビジネスロジック、データ処理 等）

### 必須チェック項目：

- **ビジネスロジックの正確性**: データの加工、計算、判定処理が仕様通りに実装されているか
- **トランザクション境界**: データベーストランザクションが適切なスコープで設定されているか
- **データ変換処理**: EntityからDTOへの変換、外部APIレスポンスの変換が適切か
- **Service間の依存関係**: 他のServiceへの依存が適切で、循環参照を避けているか
- **エラーハンドリング**: ビジネスロジック内で発生するエラーを適切に処理し、例外をスローしているか（HTTPステータスはControllerで決定）
- **責務分離**: Serviceがビジネスロジックのみに集中し、HTTP関連の処理を含んでいないか

### パフォーマンス考慮点：

- バッチ処理での一括操作
- 適切なキャッシュ戦略（TTL設定）
- 不要なDB呼び出しの削減

### 設計・命名規則：

- 特殊な実装には必ずコメントで理由を明記
- モジュール間で名前の衝突を避ける

### レビュー対象外

以下はService層ではなく、他の層でレビューすること：

- **Entity定義、データベーススキーマ** → DB層で扱う
- **TypeORMのクエリビルダーの書き方の詳細** → DB層で扱う（Serviceでは「どのデータを取得するか」のみ）
- **DTOの定義とバリデーション** → Controller層で扱う
- **HTTPリクエスト/レスポンス処理** → Controller層で扱う
- **HTTPステータスコードの決定** → Controller層で扱う
- **認証・認可ガードの設定** → Controller層で扱う
- **ルーティング、エンドポイント定義** → Controller層で扱う
